# 脚本结构图

> 本文档随 `Assets/Scripts/` 下脚本的增改而更新，用于快速查看分层与各脚本/文件夹职责。

## 分层概览

```
启动层          Services/GameBootstrap, Services/SceneManagers/TitleSceneManager
                    ↓
服务/数据层      Services/SaveManager, SaveData, AccountManager, AccountEntry, CardDataBase
                    ↓
核心逻辑层      Core/Battle (BattleManager, BattleContext, CardUnit, BattleFieldView)
                    ↓
                Core/Card (CardData, CardInstance, CardStatsService, CardUI, CardEffect)
                    ↓
UI 层           UI/BaseUIPanel, Services/UIPanelManager（含面板栈 PushPanel/Back）
                UI/TitlePanel, UI/LoginPanel, UI/SaveSelectPanel, UI/BattlePanel, UI/CardActionPanel
```

- **启动层**：场景入口，加载存档、注册卡牌；TitleSceneManager 标题场景入口显示标题面板。
- **服务/数据层**：存档读写与存档数据结构、账号注册/登录；CardDataBase 卡牌配置查表（cardId→CardData）。
- **核心逻辑层**：战斗上下文与回合制流程（BattleManager、BattleContext、CardUnit）；BattleFieldView 根据 CardUnit 生成卡牌 Prefab 并绑定 CardUI；CardUI 挂在卡牌 Prefab 上负责显示与点击。
- **UI 层**：BaseUIPanel + UIPanelManager（含面板栈，弹窗式 PushPanel/Back）；标题、登录、选档、战斗面板、卡牌操作面板。

## 目录与脚本说明

### Assets/Scripts/

项目 C# 脚本根目录，按「Core（核心玩法）」「Services（启动与存档服务）」等分层组织。

---

### Core（核心逻辑）

存放与卡牌、战斗直接相关的逻辑，不负责启动与持久化。

#### Core/Card/

卡牌相关：静态配置、运行时实例、数值计算、表现与效果。

| 脚本 | 功能 |
|------|------|
| **CardData.cs** | 卡牌静态配置（ScriptableObject）：cardId、名称、描述、攻/血/费；PVP 与 PVE 共用，是否计入强化由战斗 isPVP 决定。 |
| **CardInstance.cs** | 运行时卡牌实例（卡组/背包）：cardId、instanceId；强化字段等。 |
| **CardStatsService.cs** | 静态服务：GetAttack/GetHealth(instance, data, useEnhancement)，供 BattleManager、CardUnit 计算战力。 |
| **CardUI.cs** | 挂在卡牌 Prefab 上：Bind(CardUnit)、RefreshDisplay；点击时 PushPanel 打开卡牌操作面板。 |
| **CardEffect.cs** | 技能/效果抽象基类：Execute()，供子类实现具体效果。 |

#### Core/Battle/

战斗流程、单位与战场表现。

| 脚本 | 功能 |
|------|------|
| **BattleContext.cs** | 单局战斗只读上下文：isPVP、Player1Deck、Player2Deck。 |
| **BattleManager.cs** | 回合制驱动：BattleSide、turnCount；双方 CardUnit 列表与上场 2 张；StartBattle(ctx)、EndCurrentSideTurn(auto)、IsAllActiveUnitsActedThisTurn、GetActiveUnit、CheckBattleEnd；卡牌数据通过 CardDataBase 查询。 |
| **CardUnit.cs** | 战斗单位：由 CardInstance+Context 构造，currentHP/currentAttack、hasActedThisTurn、TakeDamage/Heal、effects。 |
| **BattleFieldView.cs** | 单例；SetUpCards 根据双方 CardUnit 生成卡牌 Prefab 并 Bind CardUI；RefreshAllCards、ClearCards。 |

### Services（服务与启动）

游戏启动入口与存档读写，依赖 Core 层。

| 脚本 | 功能 |
|------|------|
| **GameBootstrap.cs** | 场景启动入口：Awake 加载存档、向 BattleManager 注册测试卡；Start 若已登录则 SelectSlot(0)；RunTestBattleIfNeeded 已注释。 |
| **SaveData.cs** | 存档数据结构（可序列化）：playerName、gold、背包 inventory、多套卡组 decks（DeckSlot）、currentDeckIndex。 |
| **SaveManager.cs** | 按账号+槽位存档（Save_{accountId}_{slot}.json）；Load/Save、SelectSlot、HasSaveInSlot、GetCurrentDeck；Awake 时 Load。 |
| **AccountEntry.cs** | 账号条目（userName、accountId、password）与 AccountListData（账号列表）。 |
| **AccountManager.cs** | 本地账号：accounts.json、LoadAccounts、Register、Login、Logout、GetCurrentAccountId、IsLoggedIn。 |
| **CardDataBase.cs** | 单例；卡牌配置表 cardId→CardData，GetCardData、RegisterCardData。 |
| **UIPanelManager.cs** | 单例面板管理器：RegisterPanel、ShowPanel(name)、GetPanel；面板栈 useStack、PushPanel、Back、ReplacePanel、ClearStackAndShow（弹窗式，不隐藏底层面板）。 |
| **TitleSceneManager.cs** | 标题场景入口：Start 时 ShowPanel(UIPanelNames.Title)。 |

---

### UI（用户界面）

标题、登录、存档选择等界面，通过 BaseUIPanel + UIPanelManager 统一管理显隐与切换。

| 脚本 | 功能 |
|------|------|
| **BaseUIPanel.cs** | 所有 UI 面板的抽象基类：PanelName、RegisterPanel、ShowPanel、HidePanel；Awake 中向 UIPanelManager 注册并隐藏。 |
| **TitlePanel.cs** | 标题界面：登录→切 Login 面板、登出→AccountManager.Logout、设置（预留）、退出游戏。 |
| **LoginPanel.cs** | 登录界面：账号/密码 TMP_InputField、AccountManager.Login；成功→SaveSelect，取消→Title。 |
| **SaveSelectPanel.cs** | 存档选择界面：container 下槽位按钮绑定；HasSaveInSlot 为 true 时 SelectSlot 并 LoadScene("GameScene")。 |
| **BattlePanel.cs** | 战斗界面：结束回合按钮→EndCurrentSideTurn(false)；卡 1/卡 2 按钮（待接 CardActionPanel）。 |
| **CardActionPanel.cs** | 卡牌操作面板：释放技能、切换卡牌等（当前占位，待接 CardUnit 与逻辑）。 |
