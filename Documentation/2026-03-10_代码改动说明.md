# 代码改动说明 (2026-03-10)

> 逐一说明每处代码改动的**作用**、**改动前**、**改动后**、**目的**。改动清单与工作目的见 `2026-03-10_项目改动清单.md`、`2026-03-10_工作目的.md`。

---

## 1. BaseUIPanel（新建）

- **作用**：作为所有 UI 面板的抽象基类，提供注册、显示、隐藏的通用逻辑，子类只需实现或绑定按钮等具体行为。
- **改动前**：无统一面板基类，各界面若需切换需手写 SetActive 与引用管理。
- **改动后**：抽象类继承 MonoBehaviour；GetPanelName() 默认返回 gameObject.name；Awake 中调用 RegisterPanel() 并 HidePanel()；RegisterPanel() 通过 UIPanelManager.Instance 注册；ShowPanel()/HidePanel() 使用 SetActive(true/false)；预留 OnShow()/OnHide() 虚方法。
- **目的**：统一面板生命周期与显隐方式，便于 UIPanelManager 集中管理，后续新增面板只需继承并保证 GameObject 命名与 ShowPanel 参数一致。

---

## 2. UIPanelManager（新建）

- **作用**：集中持有所有已注册的 BaseUIPanel，按名称切换当前显示的面板，其余隐藏。
- **改动前**：无统一入口管理多面板显隐。
- **改动后**：单例 Instance；Dictionary&lt;string, BaseUIPanel&gt; 存储面板；RegisterPanel(name, panel) 校验后存入并调用 panel.HidePanel()；ShowPanel(name) 先遍历全部 HidePanel()，再取目标面板调用 ShowPanel()；GetPanel(name) 便于按需获取引用。
- **目的**：单一职责管理面板切换，避免各面板互相引用或散落 SetActive 逻辑。

---

## 3. TitlePanel（新建）

- **作用**：标题界面，提供登录、登出、设置、退出游戏四个入口。
- **改动前**：无标题界面脚本。
- **改动后**：继承 BaseUIPanel；OnClickLogin 调用 UIPanelManager.Instance.ShowPanel("Login")；OnClickLogout 在 accountManager 非空时调用 Logout()；OnClickSetting 预留；OnClickExit 编辑器下停播、打包下 Quit。
- **目的**：将账号与流程入口暴露到 UI，与 AccountManager 和后续 Login/SaveSelect 面板串联。

---

## 4. LoginPanel（新建）

- **作用**：登录界面，输入账号密码并调用 AccountManager.Login，成功则进入存档选择界面。
- **改动前**：无登录界面脚本。
- **改动后**：继承 BaseUIPanel；userNameInput、passwordInput 取文本；OnClickLogin 校验 accountManager 非空后 Login，失败则 return，成功则 ShowPanel("SaveSelect")；OnClickCancel 切回 ShowPanel("Title")。
- **目的**：完成「输入账号密码 → 验证 → 进入选档」的 UI 流程，与 AccountManager 对接。

---

## 5. SaveSelectPanel（新建）

- **作用**：存档选择界面，展示当前账号的存档槽位，点击已有存档的槽位后选档并进入游戏场景。
- **改动前**：无选档界面脚本。
- **改动后**：继承 BaseUIPanel；container 下按 childCount 与 SaveSlotCount 绑定槽位 Button，用局部变量 slotIndex 避免闭包错误；OnClickSlot 先判 saveManager 非空与 HasSaveInSlot(slotIndex)，无存档则提示并 return，有存档则 SelectSlot(slotIndex) 后 SceneManager.LoadScene("Game")。
- **目的**：与 SaveManager 的槽位与选档逻辑一致，仅允许有存档的槽位进入游戏，避免误建默认存档。

---

## 6. 脚本结构图（修改）

- **作用**：文档中反映当前脚本分层与各目录/脚本职责，便于查阅与 onboarding。
- **改动前**：无 UI 层与 UI 目录说明。
- **改动后**：分层概览中增加 UI 层（BaseUIPanel、UIPanelManager、TitlePanel、LoginPanel、SaveSelectPanel）；目录与脚本说明中增加「UI」小节及上述脚本表格。
- **目的**：与总结文档一致，方便后续维护与新人理解项目结构。
