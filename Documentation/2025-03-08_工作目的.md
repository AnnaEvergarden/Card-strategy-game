## 2025-03-07 工作目的

- **建立总结与恢复上下文的规范**：项目尚无统一总结方式，新建 Documentation 下的 AI_SUMMARY、USER_SUMMARY 及 `.cursor/rules/summary-docs.mdc`，目的是让后续对话能通过读取总结快速恢复「项目状态、已做修改、技术决策、待办」，并约定 USER_SUMMARY 需按文件与具体改动详细书写，避免笼统描述。
- **只读梳理、未改代码**：本轮仅阅读并归纳各脚本职责与架构，不修改任何游戏逻辑，目的是摸清当前卡牌战斗、存档、启动流程与缺失类型（如 CardInstance、CardStatsService），为后续迭代或总结提供准确基线。

## 2025-03-08 工作目的

- **支持多账号与多槽位、多卡组**：新建 AccountEntry、AccountManager 与改造 SaveData/SaveManager，目的是实现按账号隔离存档（路径含 accountId）、多存档槽位（如 3 槽）、多套卡组与当前选中卡组，为登录/选档/选卡组等玩法打基础。
- **统一调试日志与中文格式**：在所有相关脚本的 Debug 输出前加 `[类名]` 前缀，并将注释、Header、Tooltip、字符串与 Debug 文案统一为 UTF-8 简体中文，目的是便于排查问题、团队协作与避免乱码；新增 code-chinese-format.mdc 规则以约束今后代码。
- **修复槽位与当前卡组下标混用**：SaveManager 原用「当前卡组下标」当「存档槽位」决定路径与 SelectSlot，导致存读错文件；改为用独立 `currentSlotIndex` 表示槽位、`currentDeckIndex` 仅表示当前卡组，目的是保证「选第几号存档」与「选第几套卡组」语义分离、行为正确。
- **修正账号与注册逻辑**：AccountManager 启动时未加载账号列表、IsLoggedIn 返回值写反、Register 在循环内重复添加；修正后目的是确保启动即加载 accounts.json、登录状态判断正确、注册不重复添加账号。
- **提高健壮性**：GameBootstrap 在调用 AccountManager 前增加空检查，BattleManager 修正 GetDeckTotalPower 方法体笔误（`};` → `{`），目的是避免空引用与编译/运行错误。
