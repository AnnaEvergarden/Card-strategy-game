# 代码改动说明 (2025-03-08)

> 逐一说明每处代码改动的**作用**、**改动前**、**改动后**、**目的**。2025-03-07 无代码改动（只读梳理）。改动清单与工作目的见 `2025-03-08_项目改动清单.md`、`2025-03-08_工作目的.md`。

## 1. SaveManager：存档路径与槽位

### 1.1 存档路径（SavePath / GetSavePath）

- **作用**：决定存档文件路径，用于 Load/Save。
- **改动前**：路径依赖 `current.currentDeckIndex`（当前卡组下标），与「存档槽位」混用，导致选不同卡组时误读误写不同槽位文件。
- **改动后**：路径改为 `GetSavePath(currentSlotIndex)`，仅与当前选中的**存档槽位**有关，与当前选中的卡组下标无关。
- **目的**：槽位表示「第几个存档」，卡组下标表示「第几套卡组」，二者分离，避免存读错文件。

### 1.2 SelectSlot

- **作用**：切换当前使用的存档槽位并加载该槽位数据。
- **改动前**：设置 `current.currentDeckIndex = _slotIndex` 再 Load，槽位与卡组下标混用；边界为 `_slotIndex > SaveSlotCount`。
- **改动后**：设置 `currentSlotIndex = _slotIndex` 再 `Load()`；边界改为 `_slotIndex >= SaveSlotCount`。
- **目的**：用独立字段表示槽位，语义正确；边界改为 >= 与 0-based 一致。

### 1.3 HasSaveInSlot

- **作用**：判断某槽位是否有存档。
- **改动前**：边界为 `_index > SaveSlotCount`，与槽位数量语义不一致。
- **改动后**：边界改为 `_index >= SaveSlotCount`；日志文案统一为「不符合标准!」。
- **目的**：与其他槽位 API 边界一致，避免误判。

### 1.4 SetCurrentDeckIndex

- **作用**：设置当前选中的卡组下标并持久化。
- **改动前**：无效下标时仍执行 `current.currentDeckIndex = index` 与 `Save()`；边界为 `index > current.decks.Count`。
- **改动后**：无效时直接 `return`，不写入不保存；边界改为 `index >= current.decks.Count`。
- **目的**：防止越界与无效状态被写入存档。

### 1.5 SetDeckInstanceIds

- **作用**：为某套卡组设置 instanceId 列表。
- **改动前**：边界为 `_deckIndex > current.decks.Count`；无效时用 `Debug.Log`。
- **改动后**：边界改为 `_deckIndex >= current.decks.Count`；无效时改为 `Debug.LogWarning`。
- **目的**：边界一致；无效调用用 Warning 更易排查。

## 2. AccountManager

### 2.1 Awake 中 LoadAccounts()

- **作用**：启动时加载本地账号列表。
- **改动前**：Awake 中未调用 `LoadAccounts()`，账号列表可能为空。
- **改动后**：Awake 中调用 `LoadAccounts()`。
- **目的**：确保进入游戏时已从 accounts.json 加载账号，登录/注册逻辑可正确判断重复与当前账号。

### 2.2 IsLoggedIn()

- **作用**：返回当前是否已登录。
- **改动前**：返回 `string.IsNullOrEmpty(currentAccountId)`，即「空为 true」，与「已登录」语义相反。
- **改动后**：返回 `!string.IsNullOrEmpty(currentAccountId)`。
- **目的**：已登录时返回 true，与命名一致。

### 2.3 Register 中 accounts.Add

- **作用**：注册新账号并写入列表。
- **改动前**：`accountList.accounts.Add(new AccountEntry(...))` 写在 `foreach` 循环内，若有多处重复会添加多次。
- **改动后**：将 `Add` 移出循环，仅在遍历确认无重复后添加一次。
- **目的**：避免同一账号被重复添加。

## 3. GameBootstrap

### 3.1 日志前缀与中文

- **作用**：调试输出与 Inspector 展示。
- **改动前**：无统一前缀；部分注释/Header/Debug 为乱码或问号。
- **改动后**：所有 Debug 前加 `[GameBootstrap]`；注释、Header、字符串与 Debug 文案改为 UTF-8 简体中文（与 SaveManager 风格一致）。
- **目的**：便于从日志定位出处；避免乱码、统一协作风格。

### 3.2 引用 AccountManager 与登录后选槽位

- **作用**：启动时依赖账号状态并选择存档槽位。
- **改动前**：无 AccountManager；Start 直接跑 RunTestBattleIfNeeded。
- **改动后**：FindObjectOfType 获取 AccountManager；若已登录则 `SaveManager.SelectSlot(0)`；RunTestBattleIfNeeded 可注释。
- **目的**：支持多账号与多槽位流程，登录后默认选槽位 0。

### 3.3 Start 中 accountManager 空检查

- **作用**：调用账号相关逻辑前避免空引用。
- **改动前**：直接调用 `accountManager.IsLoggedIn()`，未挂 AccountManager 时会空引用。
- **改动后**：先判断 `accountManager == null`，为空则不再调用。
- **目的**：场景未挂 AccountManager 时也能安全启动。

## 4. BattleManager

### 4.1 日志前缀与中文

- **作用**：战斗流程的调试输出与 Tooltip。
- **改动前**：无统一前缀；部分文案乱码或问号。
- **改动后**：所有 Debug 前加 `[BattleManager]`；类注释、Tooltip、summary 与 Debug 文案改为 UTF-8 简体中文。
- **目的**：便于定位日志；显示与协作统一。

### 4.2 GetDeckTotalPower 方法体笔误

- **作用**：计算卡组总战力。
- **改动前**：方法体首行为 `};`，导致方法体未正确开始。
- **改动后**：改为 `{`。
- **目的**：修正语法，方法体正确闭合。

## 5. SaveManager 日志

- **作用**：存档相关警告输出。
- **改动前**：一处应属警告的提示用 `Debug.Log`。
- **改动后**：该处改为 `Debug.LogWarning`。
- **目的**：异常或边界情况用 Warning 更易在控制台区分。
