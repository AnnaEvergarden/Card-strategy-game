## 2026-03-10 工作目的

- **串联账号与存档流程到 UI**：在已有 AccountManager、SaveManager、GameBootstrap 基础上，补全「标题 → 登录 → 存档选择 → 进入游戏场景」的 UI 流程，使玩家能通过界面完成登录与选档，为后续正式 UI 与交互打基础。
- **抽象通用 UI 面板框架**：通过 BaseUIPanel + UIPanelManager，统一面板的注册与显隐切换；后续新增任意 UI 面板只需继承 BaseUIPanel 并注册，避免各处手写 SetActive 与面板管理逻辑。
- **保证选档逻辑与 SaveManager 一致**：SaveSelectPanel 在选择槽位时先 HasSaveInSlot 再 SelectSlot，仅在有存档的槽位才进入游戏场景，避免误建默认存档或进入空数据场景。
